# Implementation Plan: SensorTelemetry System

### 1. Краткое описание задачи

Разработка консольного программного обеспечения на языке C++ для автоматизированного сбора, валидации и анализа телеметрических данных с датчиков, хранящихся в формате JSON. Система предназначена для обработки больших массивов данных с целью выявления статистических закономерностей и технических аномалий в работе оборудования.

### 2. Цель и ожидаемый результат

**Цель:** создать надежный аналитический инструмент с интуитивно понятным интерфейсом для работы с JSON-телеметрией.
**Ожидаемый результат:**

* Консольное приложение (Dashboard) с поддержкой русского языка.
* Автоматически сформированные отчеты по показателям: Min, Max, AVG, медиана и количество аномалий.
* Наборы сгенерированных тестовых данных (эталонных и с ошибками).
* Отчет о производительности системы (Benchmarking).

### 3. Этапы и задачи реализации

Система разработана за 8 последовательных шагов:

1. **Анализ формата:** изучение структуры JSON-объектов (поля `ts`, `sensor`, `value`) и проектирование структуры данных в C++.
2. **Ввод/Валидация:** создание функций чтения файлов и модуля первичной проверки данных на наличие пропусков и корректность типов.
3. **Парсинг/Преобразование:** интеграция библиотеки `nlohmann/json` для эффективного маппинга JSON во внутренние контейнеры `std::vector`.
4. **Генерация данных:** реализация модуля для создания синтетических тестов с управляемым процентом ошибок (дубликаты меток, выход за границы диапазона).
5. **Вычисления/Алгоритмы:** разработка математического ядра для поиска медианы и детекции аномалий методом скользящего окна (Rolling Window).
6. **Вывод результатов:** проектирование пользовательского интерфейса (Dashboard) с использованием цветовой индикации и псевдографики.
7. **Тестирование/Исправления:** проведение стресс-тестов на 1000+ записей и устранение ошибок с кодировкой кириллицы (переход на CP1251).
8. **Оформление:** финальная сборка Release-версии и подготовка документации в репозитории.

### 4. План-график

Общий срок реализации проекта составил **4 дня**:

| День | Этап | Задачи |
| --- | --- | --- |
| **День 1** | Проектирование и I/O | Настройка IDE, подключение библиотек, реализация чтения JSON. |
| **День 2** | Валидация и генерация | Создание генератора тестов, проверка логической целостности данных. |
| **День 3** | Алгоритмы и UI | Реализация математической обработки и создание меню Dashboard. |
| **День 4** | Тестирование и финализация | Бенчмаркинг, исправление багов кодировки, сборка .exe файла. |

### 5. Ресурсы

* **Язык / IDE:** C++17, Microsoft Visual Studio 2022.
* **Библиотеки:** `nlohmann/json` (обработка данных), `chrono` (тайминги), `windows.h` (интерфейс).
* **Примеры файлов:** спецификация JSON-телеметрии, примеры `data_0.json`.
* **Материалы:** документация Microsoft по консольному API, гайды по алгоритмам статистического анализа.
